## Overview

The Markdown viewer renders GitHub Flavored Markdown (GFM) with full support for tables, strikethrough, task lists, autolinks, and syntax-highlighted code blocks. It provides a live preview that updates as you type, making it ideal for documentation authoring and README file editing.

## Libraries Used

The Markdown preview is built on a carefully selected stack of React-compatible libraries:

### Implementation

The component uses a plugin-based architecture to transform Markdown:

```tsx title="MarkdownPreview.tsx" {2-3}
<ReactMarkdown
  remarkPlugins={[remarkGfm]}
  rehypePlugins={[rehypeHighlight]}
  components={{
    pre: ({ node, children }: any) => (
      <CodeBlock node={node}>{children}</CodeBlock>
    ),
  }}
>
  {content}
</ReactMarkdown>
```

## Code Block Features

Code blocks are rendered with a custom `CodeBlock` component that provides enhanced functionality:

### Interactive Toolbar

Each code block includes a toolbar with:

- **Language badge** - Displays the detected language in uppercase (e.g., `TYPESCRIPT`, `PYTHON`)
- **Line count** - Shows the number of lines in the block
- **Collapse/expand button** - Toggle code visibility
- **Copy button** - One-click copy to clipboard with visual feedback

```tsx title="Code block toolbar" {5-8,11-14}
const handleCopy = async () => {
  await navigator.clipboard.writeText(rawText);
  setCopied(true);
  setTimeout(() => setCopied(false), 1500);
};

return (
  <div className="md-code-block">
    <div className="flex items-center justify-between">
      <div className="flex items-center gap-1.5">
        <Button variant="icon" onClick={() => setCollapsed((c) => !c)}>
          {collapsed ? <IconChevronRight /> : <IconChevronDown />}
        </Button>
        {lang && <span>{lang}</span>}
        <span>{lineCount} {lineCount === 1 ? "line" : "lines"}</span>
      </div>
      <Button variant="icon" onClick={handleCopy}>
        {copied ? <IconCheck /> : <IconCopy />}
      </Button>
    </div>
    {!collapsed && <pre>{children}</pre>}
  </div>
);
```

See `MarkdownPreview.tsx:56-127`

### Syntax Highlighting

Code blocks are syntax-highlighted using `highlight.js` with GitHub's theme. The theme automatically adapts to your selected appearance:

- **Dark mode** - Uses `github-dark.css` for comfortable night viewing
- **Light mode** - Uses `github.css` for bright environments

```tsx title="Theme switching" {5}
useEffect(() => {
  let link = document.getElementById(HLJS_LINK_ID) as HTMLLinkElement | null;
  if (!link) {
    link = document.createElement("link");
    link.id = HLJS_LINK_ID;
    link.rel = "stylesheet";
    document.head.appendChild(link);
  }
  link.href = isDark ? githubDarkUrl : githubUrl;
}, [isDark]);
```

## Empty State

When the editor is empty, the preview displays a helpful prompt:

```tsx title="Empty state handling" {1-5}
if (!content.trim()) {
  return (
    <div className="preview-empty">
      <p>Start typing Markdown in the editor...</p>
    </div>
  );
}
```

<Note>
The Markdown viewer has no error state. Invalid Markdown is rendered as-is, allowing you to see exactly how your content will appear even with malformed syntax.
</Note>
