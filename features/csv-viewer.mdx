## Overview

The CSV viewer is a full-featured data table built on TanStack Table v8. It provides advanced filtering with both search and SQL modes, column resizing and sorting, cell selection tracking, and a comprehensive status bar. The viewer is displayed full-width (no editor panel) for maximum data visibility.

## CSV Parsing

Files are parsed using PapaParse with robust error handling:

```tsx title="CSV parsing with error handling" {2-5}
const { headers, rows, error } = useMemo(() => {
  if (!deferredContent.trim()) return { headers: [], rows: [], error: null };
  const result = Papa.parse<string[]>(deferredContent.trim(), {
    skipEmptyLines: true,
  });
  if (result.errors.length > 0 && result.data.length === 0) {
    return { headers: [], rows: [], error: result.errors[0].message };
  }
  const [headerRow, ...dataRows] = result.data;
  return { headers: headerRow ?? [], rows: dataRows, error: null };
}, [deferredContent]);
```

## Search Mode

The default filtering mode uses debounced global search:

### Debounced Input

Search is implemented as a sub-component that owns its own state:

```tsx title="SearchInput with debouncing" {1-7}
function SearchInput({ onFilter }: SearchInputProps) {
  const [value, setValue] = useState("");
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    const id = setTimeout(() => onFilter(value), 300);
    return () => clearTimeout(id);
  }, [value, onFilter]);

  return (
    <div className="csv-search-wrapper">
      <IconSearch size={11} />
      <Input
        ref={inputRef}
        placeholder="Find"
        value={value}
        onChange={(e) => setValue(e.target.value)}
      />
    </div>
  );
}
```

See `CsvPreview.tsx:130-169`

<Info>
**Performance optimization:** The input owns its state internally, so keystrokes only re-render the tiny `SearchInput` component. The parent's `useReactTable` call never runs until the debounced value settles after 300ms.
</Info>

### Keyboard Shortcut

Press `⌘F` (Mac) or `Ctrl+F` (Windows/Linux) to focus the search input:

```tsx title="Keyboard shortcut" {2-8}
useEffect(() => {
```

## SQL Mode

Toggle to SQL mode for advanced filtering with WHERE conditions:

### Query Interface

Users type only the WHERE condition; the `SELECT * FROM csv WHERE` prefix is automatic:

```tsx title="SQL query builder" {4-5}
function SqlInput({ onRun }: SqlInputProps) {
  const [condition, setCondition] = useState("");

  function buildQuery() {
    const trimmed = condition.trim();
    return trimmed ? `SELECT * FROM csv WHERE ${trimmed}` : "SELECT * FROM csv";
  }

  return (
    <div className="csv-sql-wrapper">
      <Input
        placeholder="age > 30 AND name = 'Alice'  (⌘↵)"
        value={condition}
        onChange={(e) => setCondition(e.target.value)}
        onKeyDown={(e) => {
          if (e.key === "Enter" && (e.metaKey || e.ctrlKey)) {
            e.preventDefault();
            onRun(buildQuery());
          }
        }}
      />
      <Button variant="primary" onClick={() => onRun(buildQuery())}>
        <IconPlayerPlay size={11} />
        Run
      </Button>
    </div>
  );
}
```

See `CsvPreview.tsx:179-229`

### Query Execution

Queries are executed only on `⌘↵` or the Run button click (not real-time):

```tsx title="SQL query execution" {4-11}
const sqlResult = useMemo(() => {
  if (queryMode !== "sql") return null;
  try {
    // Register parsed CSV rows as table "csv"
    (alasql as any).tables.csv = { data };
    const rows = alasql(sqlQuery) as Record<string, string>[];
    return {
      rows: rows.map((r) =>
        Object.fromEntries(
          Object.entries(r).map(([k, v]) => [k, v == null ? "" : String(v)])
        )
      ),
      error: null,
    };
  } catch (e) {
    return { rows: null, error: (e as Error).message };
  }
}, [queryMode, sqlQuery, data]);
```

```tsx title="Line ending detection" {1-5}
const lineEnding = useMemo(() => {
```

- `CR` - Legacy Mac line endings (`\r`)

## Performance Optimizations

<Steps>
  <Step title="Input state isolation">
    `SearchInput` and `SqlInput` own their input state internally. Keystrokes only re-render these tiny components—the parent's `useReactTable` call never runs on every keystroke.
  </Step>
  
  <Step title="Deferred parsing">
    `useDeferredValue` delays parsing for large files, keeping the UI responsive during typing. The skeleton loader shows immediately.
  </Step>
  
  <Step title="Memoized queries">
    SQL queries and data transformations are wrapped in `useMemo` to prevent unnecessary recalculation.
  </Step>
</Steps>
