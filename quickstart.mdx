## Launch the App

After completing the [installation](/installation), start the development server:

```bash
cd desktop-app
bun run tauri dev
```

The app window will open with the **Empty State** screen:

<Frame>
  <img src="/images/empty-state.png" alt="File Viewers empty state" />
</Frame>

<Note>
The first launch compiles Rust dependencies and may take 2-3 minutes. Subsequent launches are under 10 seconds.
</Note>

## Opening Your First File

File Viewers supports two methods for opening files:

<Tabs>
  <Tab title="Keyboard Shortcut">
    ### Native File Picker

    Press the keyboard shortcut for your platform:

    <CodeGroup>
    ```text macOS
    ⌘ O
    ```

    ```text Windows/Linux
    Ctrl + O
    ```
    </CodeGroup>

    This opens the native OS file picker. Navigate to a `.md`, `.json`, or `.csv` file and click **Open**.

    <Note>
    The file picker is powered by Tauri's `@tauri-apps/plugin-dialog` and matches your OS theme automatically.
    </Note>
  </Tab>

  <Tab title="Drag & Drop">
    ### Drag a File into the Window

    1. Open your file manager (Finder, Explorer, Nautilus)
    2. Drag any `.md`, `.json`, or `.csv` file into the File Viewers window
    3. The file loads instantly when you release the mouse

    <Frame>
      <img src="/images/drag-drop-overlay.png" alt="Drag and drop overlay" />
    </Frame>

    <Note>
    When hovering over the window during a drag operation, you'll see a blue overlay with a "Drop file here" message.
    </Note>
  </Tab>
</Tabs>

## Exploring Format Modes

File Viewers automatically detects the file type and switches to the appropriate viewer mode:

<Steps>
  <Step title="Open a Markdown file">
    Try opening a `.md` or `.markdown` file:

    ```bash
    # Create a sample Markdown file
    echo '# Hello File Viewers\n\nThis is **bold** and this is *italic*.\n\n```js\nconsole.log("Syntax highlighting!");\n```' > sample.md
    ```

    Open `sample.md` via ⌘O or drag-and-drop.

    **What you'll see:**
    - **Left panel**: Monaco editor with Markdown syntax highlighting
    - **Right panel**: Live GitHub Flavored Markdown preview with:
      - Formatted headings, bold, italic
      - Syntax-highlighted code blocks (using `rehype-highlight`)
      - Support for tables, task lists, strikethrough (via `remark-gfm`)

    <Note>
    The preview updates in real-time as you type. Code blocks use the `github-dark` theme regardless of the app theme.
    </Note>
  </Step>

  <Step title="Switch to JSON mode">
    Click the **JSON** tab in the toolbar (or open a `.json` file):

    ```bash
    # Create a sample JSON file
    echo '{"name":"File Viewers","version":"0.3.0","features":["markdown","json","csv"],"config":{"theme":"dark","autoSave":true}}' > sample.json
    ```

    Open `sample.json`.

    **What you'll see:**
    - **Left panel**: Monaco editor with JSON syntax validation
    - **Right panel**: Interactive JSON tree viewer with expand/collapse controls

    **Try the toolbar actions:**
    - **Format**: Pretty-print the JSON with 2-space indentation
    - **Minify**: Remove all whitespace for compact output

    <Warning>
    If your JSON is invalid, the preview panel shows an error badge with the parse error message instead of the tree.
    </Warning>
  </Step>

  <Step title="Open a CSV file">
    Click the **CSV** tab (or open a `.csv` file):

    ```bash
    # Create a sample CSV file
    echo 'name,age,city,role\nAlice,28,NYC,Engineer\nBob,34,SF,Designer\nCarol,31,LA,Manager\nDave,29,NYC,Engineer\nEve,26,SF,Engineer' > sample.csv
    ```

    Open `sample.csv`.

    **What you'll see:**
    - **Full-width table** (no editor panel in CSV mode)
    - **Sortable columns**: Click any header to cycle through unsorted → ascending → descending
    - **Resizable columns**: Drag the column dividers
    - **Status bar** showing:
      - Row/column count (e.g., "5 rows × 4 cols")
      - Selected cell info (click any cell to see "2:3 (67 chars)")
      - Encoding (UTF-8) and line ending (LF/CRLF)

    <Note>
    Large CSV files show a skeleton loader while parsing. The table uses `useDeferredValue` for smooth rendering.
    </Note>
  </Step>
</Steps>

## Using CSV Advanced Features

The CSV viewer includes production-grade tools for data exploration:

<Tabs>
  <Tab title="Search Mode">
    ### Global Filter

    1. Type in the **Search** input at the top of the CSV table
    2. The filter applies after **300ms of inactivity** (debounced)
    3. Matching rows are highlighted; non-matching rows are hidden

    **Keyboard shortcut:**
    - Press `⌘F` (macOS) or `Ctrl+F` (Windows/Linux) to focus the search input

    <Note>
    The search is **case-insensitive** and searches across all columns. Type `nyc` to find rows containing "NYC" in any field.
    </Note>
  </Tab>

  <Tab title="SQL Mode">
    ### Query CSV Data with SQL

    1. Click the **SQL** button in the CSV toolbar
    2. Type a `WHERE` condition (e.g., `age > 28 AND city = 'NYC'`)
    3. Press `⌘↵` (macOS) or `Ctrl+Enter` (Windows/Linux), or click **Run**

    **Example queries:**

    ```sql
    -- Filter by age
    age > 30

    -- Multiple conditions
    role = 'Engineer' AND city = 'SF'

    -- Pattern matching (use SQL LIKE syntax)
    name LIKE 'A%'
    ```

    <Note>
    You only type the **WHERE condition**. The `SELECT * FROM csv WHERE` prefix is locked and auto-prepended.
    </Note>

    **How it works:**
    - Your CSV data is registered as an in-memory table named `csv` (powered by **alasql**)
    - Column names are auto-detected from the first row
    - If the query fails, a red error bar appears below the toolbar with the error message

    <Warning>
    SQL mode executes queries **on-demand** (not live). Press ⌘↵ or click Run to execute. This prevents performance issues on large datasets.
    </Warning>
  </Tab>
</Tabs>

## Switching Between Formats

Each format tab maintains **independent content**:

<Steps>
  <Step title="Load a Markdown file">
    Open `sample.md` via ⌘O. The Markdown tab becomes active.
  </Step>

  <Step title="Switch to JSON tab">
    Click the **JSON** tab. The editor is empty (or shows the last JSON content you typed).
  </Step>

  <Step title="Type JSON content">
    Paste or type JSON in the editor:

    ```json
    {"message": "This is independent from Markdown"}
    ```
  </Step>

  <Step title="Switch back to Markdown">
    Click the **Markdown** tab. Your original Markdown content is still there—**unchanged**.
  </Step>
</Steps>

<Note>
The app stores per-format content in React state (`content: Record<Format, string>`). Switching tabs does **not** clear the editor.
</Note>

## Theme Customization

File Viewers supports three theme modes:

<Steps>
  <Step title="Open the theme menu">
    Click the **theme toggle button** in the top-right toolbar (sun/moon icon).
  </Step>

  <Step title="Select a theme">
    Choose from:
    - **System**: Follows your OS appearance settings (auto-switches when OS theme changes)
    - **Light**: Always light mode
    - **Dark**: Always dark mode

    <Note>
    The theme preference is stored in React state (`themePref`). Monaco Editor and JSON viewer adapt automatically.
    </Note>
  </Step>
</Steps>

## Project Structure Overview

Understanding the codebase helps when extending features:

```
desktop-app/
├── src/
│   ├── App.tsx                  # Root component (all state lives here)
│   ├── main.tsx                 # Entry point + Monaco local-bundle setup
│   ├── App.css                  # Global styles + CSS design tokens
│   └── components/
│       ├── EditorPanel.tsx      # Monaco editor wrapper
│       ├── PreviewPanel.tsx     # Format router (Markdown/JSON/CSV)
│       ├── MarkdownPreview.tsx  # react-markdown renderer
│       ├── JsonPreview.tsx      # react-json-view-lite renderer
│       ├── CsvPreview.tsx       # TanStack Table + SQL mode
│       ├── EmptyState.tsx       # Welcome screen
│       └── ui/                  # Button, Input, Textarea primitives
├── src-tauri/
│   ├── src/lib.rs               # Tauri setup + native OS menu
│   ├── tauri.conf.json          # App configuration (window size, bundle settings)
│   └── capabilities/
│       └── default.json         # Tauri permission grants (dialog, fs, etc.)
└── docs/
    ├── architecture.md          # State management, file loading, theme system
    └── components.md            # Props reference for every component
```
